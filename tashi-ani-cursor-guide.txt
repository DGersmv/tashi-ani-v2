Версия 1.0  ·  2025




Обзор и важные правила
Этот документ — пошаговый план переноса UI-концепта сайта Таши-Ани (tashi-ani-v5.html) в существующий Next.js проект. Особое внимание уделено защите личного кабинета заказчика — его нельзя затронуть в процессе работы.

Структура работы
?  Всего 10 шагов, каждый — отдельная задача в Cursor
?  Один шаг = один коммит в git
?  Кабинет трогаем только на Шаге 9 — и только CSS-переменные
?  Каждый шаг проверяется перед переходом к следующему

?  ПРАВИЛО №1: Перед началом любого шага — git commit. Если что-то сломалось — git reset --hard HEAD.
?  ПРАВИЛО №2: Никогда не давай Cursor задачу "перепиши весь проект". Только один компонент за раз.
?  ПРАВИЛО №3: Кабинет (личный кабинет заказчика) — неприкосновенная зона до Шага 9.

ШАГ 2  FullPage Scroll — основа навигации
Что делаем
Создаём главный механизм сайта — контейнер который держит все слайды и переключает их по одному. Это самый важный компонент.

Промпт для Cursor:
Создай новый файл src/components/FullPageScroll.tsx  Компонент принимает children (массив слайдов) и реализует: 1. Хранит currentIndex в useState 2. Переключение по колесу мыши (один тик = один слайд, с debounce 800ms чтобы не пролистать несколько) 3. Переключение по свайпу на touch-устройствах (минимум 40px) 4. Переключение по стрелкам ArrowUp/ArrowDown и PageUp/PageDown 5. Анимация через Framer Motion: текущий слайд уходит вверх (y: -100vh, opacity 0), новый приходит снизу (y: 100vh ? 0, opacity 1). Длительность 0.9s, easing: [0.77, 0, 0.18, 1] 6. Экспортирует через context: currentIndex и функцию goTo(n)  Каждый слайд — это div position:fixed, inset:0, width:100vw, height:100vh Не трогай ничего за пределами этого файла.

Промпт для подключения:
В файле src/app/page.tsx оберни содержимое в компонент FullPageScroll. Передай пока один пустой слайд с текстом "Тест". Убедись что страница открывается и не ломает кабинет по другим роутам.
?  Шаг 2 завершён. git commit: "feat: FullPageScroll component"

ШАГ 3  Кастомный курсор и навигационные точки
Что делаем
Добавляем два маленьких но важных компонента — кастомный курсор и точки-навигатор справа.

Промпт — курсор:
Создай src/components/CustomCursor.tsx  Два div: маленькая точка (10px, золотая #C9A96E, mix-blend-mode: difference) и кольцо (36px, border 1px золотой, полупрозрачное). Точка следует за мышью напрямую через mousemove. Кольцо следует с lerp-задержкой (коэффициент 0.12) через requestAnimationFrame. При наведении на <a> и <button> — точка 20px, кольцо 56px. Компонент рендерить в layout.tsx вне всех роутов (глобально). На мобильных (touch-устройствах) — не рендерить совсем.

Промпт — точки навигации:
Создай src/components/SlideNav.tsx  Принимает пропсы: total (число слайдов), current (текущий индекс), onGoTo (функция). Вертикальный столбик точек position:fixed, right:28px, top:50%, transform:translateY(-50%). Активная точка — золотая #C9A96E, scale 1.5. Остальные — rgba(201,169,110,0.25). В правом нижнем углу — счётчик "01 / 09" (текущий / всего). Подключи в FullPageScroll используя контекст.
